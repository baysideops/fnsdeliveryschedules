Assumptions made

Robot cycle time = 2 × elevator time + 3 min. Outbound travel (elev min) + disengage from cart (1 min) + return travel (elev min) + attach next cart (2 min). Return time equals outbound.

Attach (2 min) charged once per cycle, at the end. Robot is "available" only after the attach completes. Not double-counted.

Greedy robot assignment: each cart assigned to the earliest-free robot available at or before that cart's departure time.

Same 3 robots serve all 3 meals — periods don't overlap. Fleet size = max needed in any single period = 3.

Distribution time = census × 1 min/tray. Worker route gaps are 7–24 min between floors. These gaps must accommodate floor-to-floor transit — needs operational validation before committing to 5 workers.

Assumptions NOT made

Not modeled: elevator wait times or robot congestion at the same elevator bank, kitchen staging space conflicts, census variability on high-census days, empty cart return logistics, or robot timing safety buffers. Worker floor-to-floor travel distances not validated against the available gap times.

Analysis based on provided hospital census and elevator time data.
